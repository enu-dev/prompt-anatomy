## プロジェクト概要

雑なプロンプトを「Role / Context / Task / Format」の4ブロックに分解・構造化することで、AIから得られる回答品質を向上させる体験ツール。エンジニア・非エンジニア両方を想定。

---

## 必須機能（MUST）

### M1. ヘッダー
- プロダクト名（例：Prompt Anatomy）
- 短い説明（1行）「プロンプトに構造を与えると、AIは別人になる。」
- モード切替タブ（診断 / ビルド）

### M2. 診断モード（Diagnose）
- M2-1. テキストエリアにプロンプトを貼り付け可能
- M2-2. 「診断する」ボタンで以下を分析：
  - Role の有無・強さ（0〜25点）
  - Context の有無・強さ（0〜25点）
  - Task の有無・強さ（0〜25点）
  - Format の有無・強さ（0〜25点）
- M2-3. 4ブロックそれぞれを色付きで「○あり / △弱い / ×なし」表示
- M2-4. 合計100点満点の「品質スコア」を大きく表示
- M2-5. 「足りない要素」を具体的なアドバイスとして表示
- M2-6. 「構造化する」ボタンで自動的にビルドモードに送って4ブロックを埋める

### M3. ビルドモード（Build）
- M3-1. テンプレート選択（5種：コード生成・文章校正・アイデア発散・要約・翻訳）
- M3-2. 4つの入力エリア（Role / Context / Task / Format）
- M3-3. 各エリアにプレースホルダーとヒントを表示
- M3-4. リアルタイムプレビュー（右側または下部）
- M3-5. プレビュー形式の切り替え（XMLタグ / プレーンテキスト）

### M4. 共通機能
- M4-1. ワンクリックコピーボタン（Copied!表示）
- M4-2. レスポンシブ対応（375px / 768px / 1280px）
- M4-3. ダークテーマ基調
- M4-4. オフラインで完全動作（CDN不使用）
- M4-5. ファイル分離（HTML/CSS/JS別ファイル）

## 診断ロジック仕様（重要）

各ブロックの判定基準（FE2小山に渡す仕様）：

### Role 判定（25点）
- 「あなたは〜です」「君は〜」「〜として」等の役割キーワードを検出 → 15点
- 専門性を示す名詞（エンジニア・編集者・先生 等）→ 追加10点
- 検出ゼロ → 0点

### Context 判定（25点）
- 文字数が30字未満 → 0点
- 30〜100字 → 10点
- 100字以上 + 背景キーワード（「現在」「状況」「以下の」「前提」等）→ 25点

### Task 判定（25点）
- 動詞（書く・作る・教える・要約する・翻訳する 等）を検出 → 15点
- 具体的な対象（何を）が含まれる → 追加10点
- 動詞なし → 0点

### Format 判定（25点）
- 形式キーワード（「箇条書き」「表形式」「JSON」「マークダウン」「○文字以内」「○項目」等）→ 25点
- なし → 0点

合計100点満点。スコアに応じて：
- 80点以上：「優秀なプロンプトです」
- 50〜79点：「もう一息」
- 50点未満：「構造化すると大きく変わります」
